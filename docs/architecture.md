# 架构设计文档

## 1. 项目概述

**项目名称**: xiaodangjia (小当家)
**项目类型**: Windows桌面文件管理软件
**核心理念**: 基于脑图式无尽画布的文件管理器，通过AI Agent实现文件关联分析、自动归档、智能整理

## 2. 技术架构

### 2.1 技术选型

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| 桌面框架 | **Tauri 2.x** | Rust后端 + Web前端，轻量、高性能、内嵌WebView2 |
| 前端框架 | **React 18 + TypeScript** | 组件化UI开发 |
| 状态管理 | **Zustand** | 轻量级状态管理 |
| 画布引擎 | **React Flow (@xyflow/react)** | 脑图节点渲染与交互 |
| 代码编辑 | **Monaco Editor** | VS Code同源代码编辑器 |
| 本地存储 | **SQLite** | 结构化元数据存储 |
| 文件处理 | **Rust生态库** (walkdir, notify) | 文件系统操作、变更监听 |

## 3. 功能模块设计

### 3.1 核心模块

#### 3.1.1 画布引擎模块 (Canvas Engine)
- **职责**: 渲染脑图式文件节点、管理缩放/平移、处理节点拖拽
- **关键特性**:
  - 无尽画布，支持无限缩放 (0.1x - 5x)
  - 节点拖拽自由定位，位置持久化
  - 子节点跟随父节点移动 (层级关系)
  - 节点连线自动吸附
  - minimap导航

#### 3.1.2 文件管理模块 (File Manager)
- **职责**: 文件系统操作、格式识别、变更监听
- **核心功能**:
  - 目录导入/同步
  - 文件CRUD操作
  - 文件格式自动识别
  - 文件变更监听 (FsWatcher)

#### 3.1.3 编辑器/阅读器模块 (Viewer/Editor)
- **职责**: 根据文件类型调用对应渲染器

#### 3.1.4 数据库模块 (Database)
- **职责**: 元数据存储、节点位置持久化
- **数据模型**:

`	ypescript
interface FileNode {
  id: string;
  projectId: string;
  path: string;
  name: string;
  extension: string;
  size: number;
  createdAt: string;
  modifiedAt: string;
  tags: string[];
  parentId: string | null;
  positionX: number;
  positionY: number;
  isCollapsed: boolean;
  isDirectory: boolean;
  children: string[];
  relatedFiles: string[];
}
`

#### 3.1.5 AI Agent模块 (规划中)
- **职责**: 文件分析、关联发现、归档建议
- **Agent能力**:
  1. **文件理解**: 读取文件内容，生成向量嵌入
  2. **关联分析**: 查找相似/相关文件，建立关联索引
  3. **自动归档**: 建议文件归属目录
  4. **标签生成**: 基于内容生成描述性标签
  5. **智能搜索**: 自然语言搜索文件

## 4. 画布交互设计

### 4.1 节点设计

`
+-------------------------+
|  📄 filename.py         |
|  🏷️ tag1, tag2          |
|  📊 12.5 KB             |
|  🔗 3 related           |
+------------+------------+
            |
        连接线
`

### 4.2 拖拽行为
- 拖动节点时，子节点自动跟随移动
- 拖动结束自动保存位置到SQLite
- 支持多选拖动

### 4.3 弹出面板
- 双击节点 → 打开阅读/编辑器面板
- 右键节点 → 上下文菜单
- 面板支持多标签页

## 5. 数据流设计

`
用户操作 → 前端事件 → Tauri IPC → Rust处理 → 数据库操作 → 返回结果 → 前端更新
     │
     └─→ 画布位置更新 → SQLite持久化
     │
     └─→ 文件变更 → 触发重新扫描 → 更新节点
`

## 7. 关键实现难点

1. **大规模文件性能**: 万级文件节点渲染优化 (虚拟化、懒加载)
2. **实时同步**: 文件系统变更与画布实时同步
3. **向量检索**: 内嵌向量库的性能与准确性平衡 (规划中)
4. **跨平台构建**: Windows安装包体积优化
